ELVES Setup Guide

Get your AI agent orchestration environment running in minutes. This guide detects your platform and walks you through each step.

1
System Requirements
▶
macOS 13+ (Ventura) Required
System Preferences → General → About. Tauri v2 requires macOS 13 or later.
Xcode Command Line Tools Required
Provides git, clang, and system headers needed for Rust compilation.
xcode-select --install
Windows 10 version 1809+ Required
Settings → System → About. Requires build 17763 or later for WebView2 support.
WebView2 Runtime Required
Pre-installed on Windows 11. On Windows 10, the ELVES installer auto-downloads it. Or install manually from Microsoft.
Visual Studio Build Tools Required for source builds
Install "Desktop development with C++" workload. Required for Rust compilation on Windows.
winget install Microsoft.VisualStudio.2022.BuildTools
Windows Terminal Recommended
ELVES uses Windows Terminal for external terminal launches. Falls back to cmd.exe if not installed.
winget install Microsoft.WindowsTerminal
System libraries Required
Tauri v2 requires webkit2gtk, libappindicator, and build essentials.
sudo apt install libwebkit2gtk-4.1-dev build-essential curl wget file libxdo-dev libssl-dev libayatana-appindicator3-dev librsvg2-dev
2
Install Git
▶

Git is required for worktree-based workspaces — ELVES creates isolated branches for every task.

Git (pre-installed with Xcode CLT)
Already installed if you ran xcode-select --install. Verify:
git --version
Git for Windows Required
Download from git-scm.com or install via winget:
winget install Git.Git
Tip: During Git install, select "Git from the command line and also from 3rd-party software" to ensure it's on your PATH. ELVES auto-configures core.autocrlf and core.longpaths for worktrees.
Git 
sudo apt install git
3
Install Node.js 18+
▶

Node.js provides npm, which is used to install Claude Code and Codex CLI runtimes (and for building ELVES from source).

Via Homebrew 
brew install node
Or use a version manager like nvm, fnm, or Volta.
Via winget or direct download 
winget install OpenJS.NodeJS.LTS
Or download from nodejs.org. ELVES scans npm global, Scoop, fnm, nvm-windows, and Volta install paths automatically.
Via NodeSource or nvm 
curl -fsSL https://deb.nodesource.com/setup_lts.x | sudo -E bash -
sudo apt install -y nodejs

Verify installation:

node --version && npm --version
4
Install AI Runtimes
▶

Install at least one. Both can coexist — ELVES lets you pick per-project.

Claude Code CLI Recommended
Anthropic's AI coding agent. Requires an Anthropic API key or a Claude Pro/Team/Enterprise subscription.
npm install -g @anthropic-ai/claude-code

Verify: claude --version

Codex CLI Optional
OpenAI's coding agent. Requires an OpenAI API key.
npm install -g @openai/codex

Verify: codex --version

Windows note: npm global installs create .cmd wrapper scripts. ELVES handles this automatically — it searches for both claude and claude.cmd when resolving binaries.
5
Install Rust (source builds only)
▶
Skip this step if you're installing ELVES from a pre-built release (Homebrew, MSI, DMG). Rust is only needed to compile from source.
Rust via rustup 
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
winget install Rustlang.Rustup
Or download from rustup.rs. Requires VS Build Tools (Step 1).
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh

Verify: rustc --version && cargo --version

Install ELVES
▶

Choose your installation method:

Option A: Homebrew (recommended) 
brew install --no-quarantine --cask mvmcode/tap/elves
Option B: Direct download
Download the .dmg from GitHub Releases.
Gatekeeper: Right-click ELVES.app → Open → Open in dialog. Or run: xattr -cr /Applications/ELVES.app
Option C: Build from source 
git clone https://github.com/mvmcode/elves.git
cd elves && npm install && npm run tauri dev
Option A: MSI Installer (recommended)
Download the .msi from GitHub Releases. Auto-installs WebView2 if needed.
SmartScreen: Click "More info" → "Run anyway" (binary is not code-signed).
Option B: Portable .zip
Download the .zip from GitHub Releases, extract, and run ELVES.exe.
Option C: Build from source

7
First Launch Checklist
▶

When ELVES starts for the first time, it will:

Detect runtimes
ELVES scans your PATH for claude and codex binaries. The status bar shows green/red indicators. If both are missing, you'll see the First-Run Wizard with install hints and a re-scan button.
Create your first project
Point ELVES at a git repository on your machine. Each project gets its own workspace, memory, and configuration.
Run your first task
Type a task in the task bar (or press Cmd+KCtrl+KCtrl+K), then click "Summon". ELVES creates a worktree, spawns the agent in a live terminal, and you can watch it work in real time.
Ship or discard
When the agent finishes, click "Ship It" to push, merge, and clean up — or "Remove" to discard the worktree.

Troubleshooting
▶
"ELVES.app is damaged" / Gatekeeper block 
xattr -cr /Applications/ELVES.app
Runtime not detected (installed via nvm/Homebrew)
ELVES resolves your login shell PATH at startup. If binaries are still not found, ensure they're accessible from a new Terminal window: which claude
SmartScreen blocks ELVES
Click "More info" → "Run anyway". The binary is not code-signed (open-source project).
Runtime not detected
ELVES scans npm global (%APPDATA%\npm), Scoop, Cargo, Volta, and fnm paths. Ensure claude or codex is accessible from a new cmd window:
where claude
Long path errors in worktrees
ELVES auto-sets core.longpaths=true per worktree. If you still hit issues, enable it globally:
git config --global core.longpaths true
Mixed line endings in diffs
ELVES configures core.autocrlf=true per worktree. For global consistency:
git config --global core.autocrlf true
WebView2 missing
The MSI installer auto-downloads WebView2. If using the portable version, install from Microsoft.
"--version timed out" in runtime detection
ELVES gives each binary 5 seconds to respond to --version. If it times out, the install may be broken. Reinstall the CLI: npm install -g @anthropic-ai/claude-code
Missing webkit2gtk
See Step 1 for the full package list. The exact packages vary by distro.
